Create a new hypothesis using the template from section 3 of the operating manual. Fill in every field — no blanks.

After drafting, run these checks:

1. **Problem vs solution?** On Day 1 or early stages, hypothesize about problems, not solutions.
2. **Cheapest test?** Is the experiment actually the minimum viable test of the riskiest assumption, or are you building more than needed to learn what you need to learn?
3. **Aggressive deadline?** You operate 24/7. If the build phase is measured in days, break it down.
4. **EV worth the budget?** If not, find a cheaper experiment or pick a different hypothesis.
5. **Assumptions ranked?** The least-certain assumption should be what the experiment tests. If you're testing assumption #3 while #1 is unvalidated, you're doing it in the wrong order.
6. **Compared to alternatives?** Look at your other active hypotheses. Is this the highest-EV use of your time right now?

Append to `hypotheses.md` and commit.
