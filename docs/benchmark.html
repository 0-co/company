MCP Token Cost Benchmark

How many tokens do AI tool definitions actually cost? Measured across OpenAI, Anthropic, Google, MCP, and JSON Schema.

MCP tool definitions can consume 40,000–143,000 tokens per request. Perplexity's CTO reported 81% of their context window consumed by tool schemas alone. These numbers show what individual tools actually cost.

Key findings

1. OpenAI format is most expensive. The {"type": "function", "function": {...}} wrapper adds ~8% overhead per tool compared to MCP or JSON Schema.

2. Complex tools cost 5–10x more than simple ones. ValidatorTool (920 tokens) vs SearchTool (83 tokens). Parameter count and description length drive the cost.

3. 15 tools = ~6,000 tokens. At 50 tools, that's ~20,000 tokens of context consumed before a single user message. At 100+ tools (common MCP servers), you're past 40,000 tokens.

4. Format switching saves ~8–12%. Moving from OpenAI to JSON Schema saves ~764 tokens across 15 tools. Useful, but tool count management has larger impact.

Measure your own tools

from agent_friend import tool, Toolkit

@tool
def your_tool(param: str) -> str:
    """Your tool description."""
    return "result"

# Per-tool estimate
your_tool.token_estimate("openai")  # tokens for OpenAI format
your_tool.token_estimate("mcp")     # tokens for MCP format

# Full toolkit comparison
kit = Toolkit([your_tool, ...])
kit.token_report()
# {'estimates': {'openai': N, 'anthropic': N, 'google': N,
#                'mcp': N, 'json_schema': N},
#  'most_expensive': '...', 'least_expensive': '...',
#  'tool_count': N}

Methodology: Token estimates use len(json.dumps(schema, separators=(",",":"))) / 4 applied to the serialized tool definition in each format. This is a heuristic; actual tokenizer output varies by model. The relative comparisons between formats are consistent regardless of tokenizer.
