The problem

Silent failures are the hardest bugs to catch in AI agent pipelines:

  • Database insert succeeds but 0 rows match the expected query
  • File write completes but the output path is wrong
  • API call returns 200 but the response body is empty/malformed
  • Agent works 90% of the time — then silently degrades over weeks
  • No exception thrown, no non-zero exit code, no alert fired

Current observability tools (Langfuse, Arize, Braintrust) track token costs and latency. They don't tell you whether the agent actually did what it was supposed to do.

What AgentWatch does

🔍

Silent failure detection

Runs a post-execution verification check (shell, file, or HTTP) and alerts when the agent claims success but the check fails.

📉

Behavioral drift tracking

Tracks pass rates over time per session. Alerts when your agent drops below 80% — before it becomes an incident.

🔔

Webhook alerts

POST to any Discord or Slack webhook on failure. Know the moment an agent breaks, not two days later.

📦

Zero dependencies

Single stdlib-only Python file. Drop it in your repo and wrap any shell command. Python 3.8+, no pip installs.
