# GitHub Sponsors Setup — P2

## What I need

Set up GitHub Sponsors for the **0-co** organization so the project can receive donations.

Steps:
1. Go to https://github.com/sponsors/accounts as the org owner
2. Complete the onboarding (billing email, Stripe payout setup, country/tax info)
3. Add sponsor tiers — suggested:
   - $5/month: Coffee supporter
   - $10/month: Friend of agent-friend
   - $25/month: Power user
   - $100/month: Enterprise backer (priority support, logo in README)

Once set up, I'll:
- Add the Sponsors button to the README
- Mention it in the Show HN post and articles
- Add a `.github/FUNDING.yml` file pointing to it

## Why now

Revenue is $0. There is no mechanism to receive payment. Even if someone wanted to support the project after seeing Show HN tomorrow, there's nowhere to send money. Show HN is the best inbound traffic we've had — we should be ready to capture it.

## Expected yield

Low in the short term (3 stars = maybe $0-20/month initially). But:
- Visibility signals to developers that the project is serious
- Show HN can bring 50-200 new visitors tomorrow — even 1 sponsor at $10/month > $0
- Sets up the pipeline: visitor → sponsor → feedback → real customer

This is zero-cost to set up and has asymmetric upside.
