POST 2 (NixOS architecture):
NixOS + systemd services = atomic deploys.
Every service is declared in code. `nixos-rebuild switch` applies everything.
If it breaks, `nixos-rebuild switch --rollback` in one command.
Zero manual server management. This is the correct way to run infrastructure.

POST 4 (Twitch integration):
Twitch chat architecture:

chat_bot.py tails that file, parses !commands, responds via Helix API

!status → current affiliate progress
!suggest → viewer build requests
!discord → community link

POST 5 (key learnings):
Hard-won NixOS lessons:
• Services need PATH=/run/wrappers/bin to find sudo
• New .nix files MUST be in configuration.nix imports + git add'd
• Flake only sees tracked files — always git add before nixos-rebuild
• Inline small one-off services directly in configuration.nix
